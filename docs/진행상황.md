스마일서브의 엔지니어에서 AWS/DevOps 엔지니어로 거듭나는 **"EKS 현대화 프로젝트"**의 여정을 한눈에 보실 수 있게 정리해 드립니다. 이 내용은 나중에 포트폴리오의 **'트러블슈팅'** 세션으로 그대로 옮겨 적으셔도 아주 좋습니다.

## 🏗️ 1단계: 인프라의 코드화 (IaC)
클릭이 아닌 **코드로 기반 시설을 만드는 단계**였습니다.

* **설치한 프로그램:**
    * **Terraform:** 인프라 생성 도구
    * **AWS CLI:** AWS 계정 연결 및 제어 도구
* **작성한 파일:**
    * `provider.tf`: AWS 리전(서울) 및 연결 설정
    * `vpc.tf`: 가상 네트워크(VPC), 서브넷, 인터넷 게이트웨이 설정
    * `eks.tf`: EKS 클러스터(두뇌), 노드 그룹(몸통), ECR(이미지 창고) 설정
* **에러 및 해결:**
    * **에러:** `Reference to undeclared resource` (노드 그룹 추가 시 클러스터 리소스를 못 찾음)
    * **원인:** 파일 내용이 분절되었거나 리소스 선언 순서 문제.
    * **해결:** `eks.tf`에 클러스터와 노드 그룹 정의를 통합하여 전체 적용(`terraform apply`).

## 🐍 2단계: Python 기반 운영 API 개발
PHP에서 벗어나 **DevOps 표준 언어인 Python**으로 도구를 만든 단계입니다.

* **설치한 패키지 (Python):** `fastapi`, `uvicorn`, `kubernetes`
* **작성한 파일:**
    * `main.py`: Kubernetes API를 호출하여 Pod 리스트를 JSON으로 반환하는 비동기 서버 코드.
* **성과:**
    * `kubectl get pods`와 동일한 결과값을 웹 API(`localhost:8000/pods`)로 확인 성공.
    * **CKAD** 지식을 코드로 승화시킴.

## 🐳 3단계: 컨테이너화 및 CI/CD 구축
작성한 앱을 포장하고 **자동 배포 파이프라인**을 연결한 단계입니다.

* **설치한 프로그램/패키지:**
    * **Docker:** 앱을 컨테이너 이미지로 빌드
    * **Helm:** 쿠버네티스 패키지 관리 도구
* **작성한 파일:**
    * `Dockerfile`: Python 앱 실행 환경 정의
    * `requirements.txt`: 필요한 라이브러리 목록
    * `.github/workflows/deploy.yml`: GitHub Actions 자동화 시나리오
    * `my-api-chart/` (Helm Chart): `values.yaml` 등 배포 설정 파일
* **에러 및 해결 (중요!):**
    1.  **GitHub Push 실패:** `.terraform` 폴더(600MB+)가 용량 초과로 거부됨.
        * **해결:** `.gitignore` 추가 및 Git 기록 초기화(`rm -rf .git`) 후 재푸시.
    2.  **ECR Registry 주소 누락:** `$ECR_REGISTRY` 변수가 비어서 빌드 실패.
        * **해결:** YAML 파일에 `id: login-ecr`를 지정하고 outputs 값을 명시적으로 참조하도록 수정.
    3.  **Python 버전 충돌:** `a2a-sdk` 등이 Python 3.10+을 요구함.
        * **해결:** Dockerfile 베이스 이미지를 `3.9-slim`에서 `3.11-slim`으로 업그레이드.
